Model Integration

When models reference other models, additional configuration is required using the Build Paths tab.

These build paths are used at build time to locate generated model libray code to compile and link, not to find model files during code generation.

Parameters to Build Paths

Build Paths consist of a series of search paths, used to find other models.

Search Path Contents

Each search path consists of two parts:

  • Base Variable - an environment variable used as the basis for the search path.
  • Path - an additional path, relative to the value of the base variable, to create the path.

Each part of the search path can be omitted. It can consist of a Base variable only, or a search path only.

Default Search Paths

The model file can be selected using the Browse button next to the Model File text field. Navigate to the desired .opendds model file and hit OK.

Additional search paths can be added through an existing search path's context menu, selecting Add Sibling.

Should the last build path be deleted from a model file, use the Outline view, navigate to searchLocations and select Add Child in its context menu.
